# План интеграции Clerk в rn-gallery-26

## Шаг 1: Регистрация и настройка Clerk
- Создать аккаунт на [clerk.com](https://clerk.com)
- Создать приложение в Clerk Dashboard
- Включить **Native API** (раздел Native applications)
- Настроить провайдеры авторизации (email/password, phone, OAuth)
- Получить `Publishable Key`

## Шаг 2: Установка зависимостей
- `@clerk/clerk-expo` — SDK Clerk для Expo
- `expo-secure-store` — опционально (для сохранения сессии, но тогда нужен Dev Build)
- `expo-web-browser` — уже установлен ✅
- `expo-linking` — уже установлен ✅

## Шаг 3: Настройка переменных окружения
- Создать `.env` файл
- Добавить `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY`
- Настроить `.gitignore` для `.env`

## Шаг 4: Настройка Deep Linking
- В `app.json` уже есть `"scheme": "rngallery26"` ✅
- Настроить redirect URL в Clerk Dashboard
- Настроить linking config для OAuth flow

## Шаг 5: Обёртка приложения в ClerkProvider
- Обновить `app/_layout.tsx`
- Обернуть в `<ClerkProvider>`
- Настроить `tokenCache` (в памяти для Expo Go)
- Обработка состояния инициализации

## Шаг 6: Создание экрана входа
- Создать `/login` — форма входа
- Email/password или Phone
- Обработка ошибок и loading состояний

## Шаг 7: Создание экрана регистрации
- Создать `/register` — форма регистрации
- Валидация полей
- Подтверждение email/phone (если нужно)

## Шаг 8: Защита роутов (Route Guards)
- Создать layout для защищённых роутов
- Проверка `useUser()` или `useAuth()`
- Редирект на `/login` если не авторизован
- Loading экран во время проверки сессии

## Шаг 9: Интеграция с UI
- Показывать имя/email пользователя
- Кнопка выхода (`useSignOut()`)
- Обновить навигацию (скрыть login если авторизован)
- Аватар пользователя (если есть)

## Шаг 10: Обработка состояний
- Loading (пока инициализируется Clerk)
- Error (ошибка сети, неверные данные)
- Session expired (автоматический рефреш)
- Offline режим

## Шаг 11: Тестирование в Expo Go
- Запустить `npx expo start`
- Отсканировать QR-код в Expo Go
- Протестировать вход/регистрацию
- Проверить сохранение сессии (⚠️ в памяти — пропадёт при перезапуске)
- Протестировать выход

## Шаг 12: Подготовка к production (опционально)
- Добавить `expo-secure-store` для сохранения сессии
- Собрать Dev Build: `npx expo run:android`
- Включить нативный OAuth (Google/Apple sign-in)
- Настроить биометрию (если нужно)
