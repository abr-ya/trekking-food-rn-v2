# План интеграции Better Auth в rn-gallery-26

## Шаг 1: Установка зависимостей
- `@better-auth/expo` — клиент Better Auth для Expo
- `expo-secure-store` — хранение сессии (токены)
- Проверить совместимость версий с Expo SDK 54

## Шаг 2: Создание клиента Better Auth
- Создать файл `lib/auth-client.ts` (или `src/auth/`)
- Инициализировать `createAuthClient()` с URL вашего сервера
- Настроить `expoPlugin` для хранения сессий

## Шаг 3: Настройка Deep Linking
- Better Auth использует OAuth через WebView/браузер
- После авторизации нужен redirect обратно в приложение
- `expo-linking` уже установлен ✅ — нужно настроить схему URL
- В `app.json` уже есть `"scheme": "rngallery26"` ✅

## Шаг 4: Экраны авторизации
- Создать `/login` — форма входа (email/password или OAuth)
- Создать `/register` — регистрация
- Или добавить в существующий tab

## Шаг 5: Защита роутов (Route Guards)
- Создать middleware/layout для защищённых роутов
- Проверять сессию при загрузке
- Редирект на `/login` если не авторизован

## Шаг 6: Интеграция с UI
- Показывать имя пользователя / аватар
- Кнопка выхода
- Обновить навигацию (скрыть login если авторизован)

## Шаг 7: Обработка состояний
- Loading (пока проверяется сессия)
- Error (ошибка сети, неверные данные)
- Refresh токена (автоматически через `@better-auth/expo`)

## Шаг 8: Пересборка Dev Build
- После установки `expo-secure-store` нужно:
  ```
  npx expo run:android
  ```
- Hot Reload продолжит работать ✅

## Шаг 9: Тестирование
- Вход по email/password
- OAuth (если используете)
- Сохранение сессии после перезапуска приложения
- Выход и повторный вход

## Что нужно от разработчика:

1. **URL Better Auth сервера** (например `https://api.example.com`)
2. **Какие провайдеры используются?** (email/password, Google, GitHub, и т.д.)
3. **Какие роуты должны быть защищены?** (все, кроме login?)
4. **Есть ли уже UI для логина** или создать с нуля?
